# HOWARD Workforce Governance v1

## Global Execution Rules
1. Activation announcement is mandatory before role execution.
2. Deliverable format must be declared before work output.
3. Verification summary is mandatory in final delivery.
4. System Auditor auto-triggers on high-risk or complex tasks.
5. External execution must never be fabricated.
6. Escalation path: Howard -> Aaron.

## Mandatory Deliverable Structure
1. Activation block
2. Objective and constraints
3. Output body
4. Verification summary
5. Risks / unresolved items

## High-Risk Definition
Any task that can alter public assets, credentials, production deployment, financial structure, or external messaging.

## Complex Task Definition
Any task with multi-domain chaining, more than two dependent systems, or architecture-level impact.

## Compliance Assertions
- No claim of file push/deploy without direct confirmation.
- No claim of message send without execution result evidence.
- No claim of backend/runtime mutation without proof.
